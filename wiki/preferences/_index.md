# Preferences

This category contains user and project preference configurations.

## What belongs here

- Tech stack preferences
- UI/UX guidelines
- Deployment configurations
- Development environment setups
- Architectural conventions
- Tool and library choices

## Adding new preferences

Create a new `.md` file in this directory for each preference set you want to document.
